# ทำ via สวยๆ วันแข่ง (เมื่อ waypoint สนามจริงเปลี่ยน)

ตำแหน่งผู้ประสบภัย/ตำแหน่ง tag ประกาศ + เปลี่ยนได้วันแข่ง → ต้องแก้ `nav_waypoints.yaml` เร็ว
ขั้นตอนนี้ทำให้ได้เส้นทาง via สวย+ปลอดภัยแบบเดิมในไม่กี่นาที

## 3 ขั้นตอน (pick_waypoints.py → gen_via.py → ดูรูป)

### 1. คลิกเก็บ waypoint + via บนแผนที่
```bash
python3 pick_waypoints.py
```
- **คลิกซ้าย ×2** = waypoint (คลิก1=ตำแหน่ง, คลิก2=ทิศหุ่น) → เลือกปุ่ม Person/Tag/HOME ก่อนคลิก
  - **Person/Tag: ทิศคลิกให้ดี** = หันเข้ากระเป๋า / หันหา tag (gen_via ไม่แตะทิศ wp ภารกิจ)
  - **HOME: ทิศคลิกหยาบได้** — gen_via ตั้งทิศ HOME ให้หันตามทางที่วิ่งเข้ามาเอง
- **คลิกขวา ×2** = via (คลิก1=ตำแหน่ง, คลิก2=ทิศ) → via ผูกกับ waypoint ที่คลิกถัดไป
  - **ทิศ via คลิกหยาบๆ ได้** เดี๋ยว gen_via จับให้เป๊ะ — สนใจแค่ "ตำแหน่ง" ให้อยู่ในช่องว่าง + มุมฉาก
- กด **Save & Quit** → เซฟลง `nav_waypoints.yaml` (สำรอง `.bak` ให้)

### 2. รัน gen_via.py จัดฉาก + จับทิศ + เช็ค + วาด (คำสั่งเดียวจบ)
```bash
python3 gen_via.py
```
มันจะ:
1. **จัด via ให้ฉาก** — via ที่เยื้องกัน ≤6cm ในแกน x/y ดึงให้ตรงเป๊ะ (คลิกหยาบๆ ได้ เดี๋ยวจัดให้)
2. **เขียนทับทิศ via** ให้หันไปจุดถัดไปเป๊ะ
3. **ตั้งทิศ HOME** ให้หันตามทางที่วิ่งเข้ามา (waypoint ที่ไม่มี type) — wp ภารกิจ (person/tag) ไม่แตะ
4. **เช็คระยะห่างกำแพง** เตือน via/HOME ที่ชิด (<0.18m)
5. **วาด** `docs/wp_overlay.png`

> ⚠️ บางครั้งการจัดฉากดึง via ไปชิดกำแพง (เช่น snap x ให้ตรง waypoint ที่อยู่ติดผนัง) — ถ้าเจอ "ชิดกำแพง!" ให้ขยับ waypoint/via นั้นออก แล้วรันซ้ำ

### 3. เปิดรูปดู — ถ้าเส้นไม่สวย/ชน กลับไปคลิกใหม่ใน pick_waypoints หรือแก้ x,y แล้วรัน gen_via ซ้ำ
```bash
xdg-open docs/wp_overlay.png
```

> เคล็ดมุมฉาก: ตอนคลิก via ใน pick_waypoints ให้ตำแหน่ง via ที่ติดกัน **อยู่แนวตั้ง/แนวนอนเดียวกัน** (เล็ง x หรือ y ให้ตรงกัน) — status bar ล่างจอโชว์ค่า world x,y ตอนเลื่อนเมาส์ ใช้เล็งได้

## กฎ via สวย (cheat-sheet)

| กฎ | ทำยังไง |
|---|---|
| **ทิศ via** | ปล่อยให้ `gen_via.py` เติม (หันไปจุดถัดไปเสมอ) |
| **มุมฉาก** | ให้ via ที่ติดกัน **share x หรือ y** เช่น `(1.5,-0.52)→(1.5,-0.92)` = ดิ่งตรง |
| **เข้ากระเป๋า** | via จุดสุดท้ายก่อนถึง wp อยู่**หน้าด้านที่เปิด** ของกระเป๋า |
| **เดินในที่โล่ง** | อย่าให้เส้นลากตัดมุม/ทะลุ obstacle — แทรก via ในช่องว่าง |
| **ห่างกำแพง** | via + HOME ต้องห่าง ≥ 0.18m (`gen_via.py` เตือนเอง) |
| **เลี่ยงช่องแคบ** | ถ้าช่องแคบ <0.4m อ้อมทางกว้างแทน (เช่น ช่องกลางสนาม y≈-0.9) |

## โครงสนามปัจจุบัน (อ้างอิงตอนวาง via)
- กระเป๋า/obstacle: U บนซ้าย (เปิดบน), ⊐ บนขวา (เปิดซ้าย), E ล่างขวา (เปิดขวา), U ล่างซ้าย (เปิดล่าง)
- ช่องว่างกว้าง: แถบกลาง y≈-0.9 (ลอดข้าม E ได้ปลอดภัย), ช่องซ้าย x≈0.2, ขอบล่าง y≈-2.25
- map: origin (-0.387,-2.54) res 0.05 — ถ้า re-SLAM แล้ว origin เปลี่ยน ต้องแก้ค่าใน `gen_via.py`/`draw_waypoints.py`

## ไฟล์เครื่องมือ
- `pick_waypoints.py` — คลิกเก็บ wp+via บนแผนที่ (Tkinter) → เซฟ `nav_waypoints.yaml`
- `gen_via.py` — จับทิศ via ให้เป๊ะ + เช็คกำแพง + วาด (รันหลัง pick ทุกครั้ง)
- `draw_waypoints.py` — วาด overlay เฉยๆ: `python3 draw_waypoints.py [yaml] [out.png]`
- `get_waypoint.py` — บันทึก waypoint จาก TF จริง (ทางเลือก ถ้าอยากเก็บจากหุ่นจริงแทนคลิก)
